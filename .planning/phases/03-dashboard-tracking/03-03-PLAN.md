# 📐 Calibration — Medianas por Tipo de Tarefa

**Status:** 🌱 SEED (ainda vazio — Phase 4 popula automaticamente quando comando `/loga` existir e houver >= 15 amostras por tipo).

> Este arquivo é a **memória quantitativa** do coach. Quando ele vai estimar "quanto tempo vai levar essa tarefa?", consulta esta tabela. **Sem >= 3 amostras de um tipo, coach responde "sem histórico suficiente" — nunca chuta** (regra anti-alucinação do [[../PROJECT|PROJECT.md]]).

---

## 1. Propósito

Agregar `tarefas_log[]` (do frontmatter de todos os dailies) em medianas por `tipo`. Coach usa isso para:

- **Estimar tempo** de tarefa nova: "você vai demorar ~45min baseado em 12 escritas anteriores com mediana 45min"
- **Detectar viés**: "você subestima escritas em ~30min consistentemente — quer recalibrar?"
- **Negar estimativa** se N < 3: "só tenho 2 amostras de 'código', sem histórico suficiente"

---

## 2. Schema da tabela

| Coluna | Tipo | Origem |
|--------|------|--------|
| `tipo` | string | Categoria do tarefas_log (escrita, codigo, reuniao, leitura, exercicio, admin, outro) |
| `amostras` | int | Quantidade de entradas `tarefas_log` com esse tipo + `concluida: true` + `tempo_real != null` |
| `mediana_estimado` | int (min) | Mediana de `tempo_estimado` nas amostras |
| `mediana_real` | int (min) | Mediana de `tempo_real` nas amostras |
| `bias_min` | int | `mediana_real - mediana_estimado`. Positivo = subestima (leva mais que pensa). Negativo = superestima. |
| `bias_pct` | number | `bias_min / mediana_estimado * 100`. Threshold de 20% → coach alerta "viés sistemático". |
| `ultima_atualizacao` | YYYY-MM-DD | Data da última amostra incluída no cálculo. |

# ⏱️ Time Logging — 3 níveis de registro

Como o sistema captura **onde você gasta seu tempo**. Existem 3 níveis, com fricção crescente e valor crescente para o coach.

---

## 1. Visão geral dos 3 níveis

| Nível | Método | Fricção | O que gera | Usado por |
|-------|--------|---------|-----------|-----------|
| 🟢 **Auto** | Pomodoro Timer plugin | Zero (clica start/stop) | `pomodoros: N` no frontmatter | [[../00 - Dashboard/Semana-Atual\|Semana-Atual]], [[../00 - Dashboard/Mes-Atual\|Mes-Atual]] |
| 🟡 **Manual livre** | Seção `## 📊 Log de Tempo` no daily | Baixa (texto livre) | Linha de prosa (não estruturado) | Releitura humana, leitura qualitativa pelo coach |
| 🔴 **Estruturado** | Comando `/loga` (Phase 4) OU edição direta do frontmatter | Média (Claude parseia ou YAML) | `tarefas_log[]` objeto com `tempo_estimado`, `tempo_real`, `tipo` | [[calibration\|_meta/calibration]] (Phase 4+), coach para estimativas |

**Regra prática:** quanto mais você loga no nível 🔴, mais o coach calibra estimativas. Até 15 amostras por tipo = "sem histórico suficiente". Depois de 15 = estimativa com IC de ±15%.

## 3. Nível 🟡 — Log de Tempo manual (prosa)

A seção `## 📊 Log de Tempo` do template diário ([[../Templates/TPL-Diario|TPL-Diario]] linha ~89) é texto livre. Convenção:

```markdown
## 📊 Log de Tempo

- 09:00–10:30 → código feature X | estimei 2h | real 1h30
- 10:30–11:00 → email + inbox zero | estimei 30min | real 30min
- 14:00–15:45 → escrita doc arquitetura | estimei 1h | real 1h45
```

**Vantagens:**
- Fricção mínima (só texto)
- Preserva contexto (pode adicionar tags, menções, links)

**Desvantagens:**
- Não é estruturado — Dataview não agrega
- Não alimenta `_meta/calibration.md`
- Sujeito a typos e inconsistências ("código" vs "codigo" vs "code")

**Quando usar:** dias em que você só quer um registro leve. Não exige cálculo de bias.

# 🔌 Tracking Integration — Pomodoro Timer → Frontmatter

Como o plugin **Pomodoro Timer** (eatgrass/pomodoro-timer, v1.2.3 instalado em Phase 1) escreve `pomodoros: N` no frontmatter do daily ativo.

---

## 1. Arquitetura de tracking (resumo)

```
┌─────────────────────────────────────────────────────────────┐
│ USER clica START em Pomodoro Timer (sidebar right)          │
└──────────────────────────┬──────────────────────────────────┘
                           │

└──────────────────────────┬──────────────────────────────────┘
                           │ (25 min rodando…)
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ Timer chega a zero OU usuário clica STOP                     │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ Plugin abre o arquivo marcado, lê frontmatter                │
│   - Se `pomodoros: N` existe → incrementa para N+1           │
│   - Se não existe → adiciona `pomodoros: 1`                  │
└─────────────────────────────────────────────────────────────┘
```

## 2. Configuração do Pomodoro Timer plugin

### Passos (uma vez só)

1. Abrir **Settings** (Ctrl+,) → **Community plugins** → **Pomodoro Timer**
2. Na aba **General**:
   - ✅ **Auto-start focus timer**: conforme preferência
   - ⏱ **Focus duration**: 25 (default)
   - ⏱ **Break duration**: 5 (default)
3. Na aba **Logging / Frontmatter** (se disponível — depende da versão):
   - ✅ **Write to frontmatter**: **ATIVAR**
   - 📝 **Frontmatter property name**: `pomodoros` (exato — matches schema.md)
   - 🎯 **Log target**: `Active file` (default — escreve no arquivo ativo quando timer start)

**Se a aba "Logging / Frontmatter" NÃO existe** na versão 1.2.3:
- Essa feature pode estar em aba "Experimental" ou "Advanced" — procure `frontmatter` na busca da config
- Fallback: plugin **Simple Time Tracker** oferece a mesma feature (ver [[plugins]] para migração)

### Captura pré-requisito: daily ativo

Para o plugin escrever em `pomodoros:`, o daily de hoje precisa ser o arquivo ativo QUANDO você clica START. Fluxo correto:

1. Abrir daily: **Ctrl+P → Periodic Notes: Open today's daily note**
2. Clicar START no Pomodoro Timer (sidebar)
3. Trabalhar 25 min
4. Abrir outros arquivos durante o pomodoro é OK — plugin lembra do arquivo original

## 4. Troubleshooting

### Problema: Pomodoro finaliza mas `pomodoros:` não incrementa

**Causa 1:** Config "Write to frontmatter" não está ATIVA.
**Solução:** Settings → Pomodoro Timer → Frontmatter → ativar.

**Causa 2:** Plugin está escrevendo em propriedade com nome diferente (ex: `pomos` em vez de `pomodoros`).
**Solução:** Settings → Pomodoro Timer → Frontmatter property name → `pomodoros`.

**Causa 3:** Daily NÃO estava ativo no momento do START — plugin escreveu em outro arquivo (talvez em `Home.md`!).
**Solução:** Sempre certifique-se que o daily está em foco antes de START. Se poluiu outro arquivo, remova manualmente o `pomodoros:` errante.

**Causa 4:** Versão do plugin (1.2.3) não tem essa feature.
**Solução:** Migrar para TaskNotes ou Simple Time Tracker (ver [[plugins]] fallbacks).

### Problema: Incrementa mas o dashboard ainda mostra 0

**Causa:** Dataview cache stale.
**Solução:** Ctrl+P → **Dataview: Rebuild current view** (ou fechar e reabrir o arquivo do dashboard).

### Problema: YAML quebrado ("! Invalid YAML" no daily)

**Causa:** Plugin escreveu `pomodoros:` com valor inválido (string em vez de número).
**Solução:** Editar manual para `pomodoros: N` (número). Reportar bug se repetir.

- **Nível 🟢 Pomodoro** (este doc) → alimenta `pomodoros:` field (quantitativo bruto)
