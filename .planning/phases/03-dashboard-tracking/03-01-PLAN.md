## 🎯 HOJE

```dataviewjs
// Tenta abrir daily de hoje. Se não existe → empty state graciosa.
const hoje = moment().format("YYYY-MM-DD");
const mesFolder = moment().format("MM-MMMM");
const anoFolder = moment().format("YYYY");
const path = `01 - Diário/${anoFolder}/${mesFolder}/${hoje}`;
const daily = dv.page(path);

if (!daily) {
  dv.paragraph(`_Ainda sem daily para hoje (${hoje})._`);
  dv.paragraph("Crie pelo menu: **Ctrl+P → Periodic Notes: Open today's daily note**");
} else {
  // 3 prioridades
  dv.header(3, "3 Prioridades");
  const prios = daily.prioridades ?? [];
  const vazias = prios.every(p => !p || p.toString().trim() === "");
  if (vazias) {
    dv.paragraph("_Prioridades ainda não definidas._");
  } else {
    const marcados = daily.prioridades_feitas ?? 0;
    dv.list(prios.map((p, idx) => {
      const feita = idx < marcados;
      const texto = (p && p.toString().trim()) ? p : "_(vazia)_";
      return `${feita ? "✅" : "⬜"} ${texto}`;
    }));
  }

  // Contador pomodoros + humor/energia (se preenchidos)
  dv.header(3, "Sinais do dia");
  const linhas = [];
  if (daily.pomodoros != null) linhas.push(`🍅 Pomodoros hoje: **${daily.pomodoros}**`);
  if (daily.humor != null && daily.humor !== 0) linhas.push(`😊 Humor: **${daily.humor}/10**`);
  if (daily.energia != null && daily.energia !== 0) linhas.push(`⚡ Energia: **${daily.energia}/10**`);
  if (daily.sono_h != null && daily.sono_h !== 0) linhas.push(`😴 Sono: **${daily.sono_h}h**`);
  if (daily.pilar_media != null) linhas.push(`📊 Média 5 pilares: **${daily.pilar_media.toFixed(1)}**`);
  if (linhas.length === 0) {
    dv.paragraph("_Ainda não preencheu sinais hoje._");
  } else {
    dv.list(linhas);
  }

  dv.paragraph(`[[${path}|📖 Abrir daily completa]]`);
}
```

**DECISÕES DOCUMENTADAS:**

1. **Saudação via DataviewJS (não Templater)** — porque Home.md é um arquivo LIVE (não template). Templater não dispara em leitura; precisa ser DataviewJS que roda a cada open.

2. **Frase rotativa dentro do callout `> [!quote]`** — PITFALLS documenta que embed `![[]]` dentro de callout não atualiza, mas DataviewJS inline funciona bem dentro de callout. Testei a distinção.

3. **HOJE usa `dv.page(path)`** em vez de `dv.pages('"01 - Diário"').where(...)` — acesso direto é O(1) e se daily não existe retorna `undefined`, permitindo empty-state limpo.

4. **Embed `![[05 - Rotinas/Ativa]]` FORA de callout** — respeita PITFALL #12 (embed inside callout bug).

5. **Zero blocos Tracker** — DASH-01 é explícito: "**Zero Tracker charts aqui** (performance)". Mover para Plan 02 (sub-dashboards).

6. **Scoping**: dv.page() acessa 1 arquivo específico. dv.page("07 - Frases/Banco-Frases") também 1 arquivo. Nenhuma query path-scoped necessária (não há iteração).

7. **Empty-state safe**: cada bloco DataviewJS tem branch `if (!daily)` ou `if (!banco || !banco.frases)` com mensagem graciosa. Day 0 nunca renderiza erro.

8. **DASH-08 (Calendar widget)**: não precisa configurar aqui — o plugin Calendar já aparece no sidebar esquerdo por default (Fase 1 instalou). Nota de rodapé menciona para o usuário saber onde ele está.

9. **FRASE-02 (rotação dayOfYear % length)**: implementado via `moment().dayOfYear() % banco.frases.length`. Determinístico (mesmo dia = mesma frase; dia diferente = frase diferente).

10. **Atalhos**: incluem apenas itens que existem após Fase 2 (TPL-Diario, rubric, schema, _INDEX de Livros/Rotinas/Metas/Pessoas). Se algum _INDEX não existir, wikilink fica vermelho mas não quebra nada.
