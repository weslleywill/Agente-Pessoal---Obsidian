  <read_first>

  <files>
    %SystemRoot%\System32\config\SYSTEM (Registry hive — HKLM\SYSTEM\CurrentControlSet\Control\FileSystem)
  </files>
  <what-built>
    Plan 01 criou o skeleton do vault. Mas Windows 11 por padrão limita paths a 260 chars. Vault path começa em 49 chars (`E:\Claude Code\Agente Pessoal - Obsidian\Vault\`). Com daily notes + subfolders + anexos o path cresce rápido e git/Explorer quebram silenciosamente.
  </what-built>
  <how-to-verify>
    **Opção A — Registry (mais rápido, 30 segundos):**

    1. Abrir PowerShell **como Administrador** (botão direito → "Run as administrator")
    2. Executar EXATAMENTE:
       ```powershell
       reg add "HKLM\SYSTEM\CurrentControlSet\Control\FileSystem" /v LongPathsEnabled /t REG_DWORD /d 1 /f
       ```
       Output esperado: `The operation completed successfully.`
    3. **Reiniciar o computador** (a flag só aplica após reboot)
    4. Após reboot, abrir PowerShell (não precisa admin agora) e rodar:
       ```powershell
       Get-ItemPropertyValue -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" -Name LongPathsEnabled
       ```
       Output esperado: `1`

    **Opção B — Group Policy (GUI, se não gosta de CLI):**

    1. `Win+R` → digite `gpedit.msc` → Enter
    2. Navegar: **Local Computer Policy → Computer Configuration → Administrative Templates → System → Filesystem**
    3. Clicar duplo em **"Enable Win32 long paths"**
    4. Selecionar **Enabled** → Apply → OK
    5. Reiniciar computador
    6. Verificar como Opção A passo 4

    **Teste funcional (opcional, pós-reboot):**

    ```bash
    cd "/e/Claude Code/Agente Pessoal - Obsidian/Vault/01 - Diário"
    # Criar arquivo com nome intencionalmente longo
    mkdir -p "2026/04-Abril"
    touch "2026/04-Abril/2026-04-23-teste-de-path-extremamente-longo-para-verificar-que-longpaths-esta-funcionando-no-windows-11-pro-abcdefghijklmno.md"
    # Se criou sem erro "filename too long" → sucesso
    ls -la "2026/04-Abril/"
    # Deletar teste:
    rm "2026/04-Abril/2026-04-23-teste-de-path-extremamente-longo-para-verificar-que-longpaths-esta-funcionando-no-windows-11-pro-abcdefghijklmno.md"
    ```
  </how-to-verify>
  <acceptance_criteria>
    - Comando PowerShell `Get-ItemPropertyValue -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" -Name LongPathsEnabled` retorna `1`
    - Reboot foi executado após a mudança (sem reboot a flag não está ativa, mesmo que o valor esteja 1)
    - (Opcional) Arquivo teste com path >250 chars em `Vault/01 - Diário/` foi criado e deletado sem erro "filename too long"
  </acceptance_criteria>
